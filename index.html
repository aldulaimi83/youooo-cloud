Roadmap

Cloud learning roadmap

A simple path from cloud basics to real-world cloud work.

Stage 1: Foundations

  • What cloud computing is
  • Regions, availability zones, shared responsibility
  • AWS core services overview
  • Basic security mindset

Stage 2: AWS Core Services

  • EC2, S3, IAM, VPC
  • Load balancers and DNS basics
  • Monitoring with CloudWatch
  • Storage and networking concepts

Stage 3: DevOps and Deployment

  • Git workflow
  • Build and deployment thinking
  • CI/CD fundamentals
  • Environment and release discipline

Stage 4: Containers and Kubernetes

  • Docker basics
  • Pods, deployments, services
  • Configuration and scaling
  • Cluster operations mindset
Labs

Hands-on practice labs

These labs are designed to build real confidence, not just theory.

Lab 1: Launch an EC2 instance

Create a virtual server, connect to it, and understand security group basics.

Lab 2: Create an S3 static site

Host a simple site with object storage and understand public access carefully.

Lab 3: Build IAM users and roles

Practice least privilege and understand users, groups, and roles.

Lab 4: Create a VPC design

Map public/private subnets, routes, and internet access basics.

Lab 5: Containerize an app

Understand how an app is packaged into a container workflow.

Lab 6: Write a simple CI/CD flow

Connect code change to build-and-deploy thinking.

Lab 7: Deploy to Kubernetes

Practice deployments, services, and rollout ideas.

Lab 8: Monitor cloud health

Use logs and metrics thinking to understand system behavior.

Projects

Portfolio-style cloud projects

Projects are where cloud learning becomes job-ready proof.

Project 1: Static website on AWS

Host a website using S3 and connect it to a clean delivery path.

  • Storage-based hosting
  • Public delivery thinking
  • Basic DNS understanding

Project 2: Linux web server deployment

Launch a Linux server, install a web service, and expose it securely.

  • Compute basics
  • Security groups
  • Operational steps

Project 3: CI/CD demo pipeline

Show how code moves from repo to deployment using a simple pipeline idea.

  • Git-based flow
  • Automation mindset
  • Release discipline

Project 4: Container app deployment

Package an app in a container and prepare it for modern hosting.

  • Docker basics
  • Image thinking
  • Service portability

Project 5: Kubernetes app rollout

Deploy an app to a Kubernetes environment with service exposure.

  • Deployments
  • Scaling concepts
  • Service routing

Project 6: Cloud engineer portfolio stack

Combine AWS, Linux, CI/CD, and Kubernetes into one showcase project.

  • Architecture thinking
  • Operational design
  • Portfolio proof
